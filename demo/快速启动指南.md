# 🚀 摩托车服务平台 - 快速启动指南

## 📦 项目结构

```
redame/
├── demo/                    # 后端项目（Spring Boot）
└── vue-project/             # 前端项目（Vue 3）
```

## ⚡ 快速启动步骤

### 1️⃣ 启动数据库

```bash
# 确保MySQL服务运行
# Windows: 服务管理器中启动MySQL服务
# Linux/Mac: 
sudo service mysql start
# 或
sudo systemctl start mysql
```

### 2️⃣ 导入数据库

```bash
# 进入项目目录
cd F:/redame/demo

# 导入SQL脚本
mysql -u root -p motorcycle_service_platform < motorcycle_service_platform.txt
# 输入MySQL密码
```

### 3️⃣ 配置数据库密码

编辑 `demo/src/main/resources/application.properties`:

```properties
spring.datasource.password=你的MySQL密码
```

### 4️⃣ 启动后端

```bash
cd F:/redame/demo

# Windows
mvnw.cmd spring-boot:run

# Linux/Mac
./mvnw spring-boot:run
```

**等待后端启动成功**，看到以下信息:
```
Started DemoApplication in X.XXX seconds
```

### 5️⃣ 启动前端

**打开新的命令行窗口**:

```bash
cd F:/redame/vue-project

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm run dev
```

**等待前端启动成功**，看到:
```
VITE vX.X.X  ready in XXX ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

### 6️⃣ 访问应用

打开浏览器访问: **http://localhost:5173**

## 🔧 常见问题

### ❓ 后端无法启动

**问题**: 控制台报错 Connection refused 或数据库连接失败

**解决方案**:
1. 确认MySQL服务已启动
2. 检查数据库密码是否正确
3. 确认数据库 `motorcycle_service_platform` 已创建
4. 查看完整错误日志

---

### ❓ 前端无法连接后端

**问题**: 登录时显示网络错误

**解决方案**:
1. 确认后端已启动（访问 http://localhost:8080）
2. 检查浏览器控制台Network标签
3. 后端未启动时，前端会自动使用模拟数据

---

### ❓ 注册后无法登录

**问题**: 提示用户名或密码错误

**解决方案**:
1. 检查用户类型是否匹配（用户/商家/管理员）
2. 查询数据库确认用户是否成功创建:
   ```sql
   SELECT * FROM sys_user ORDER BY create_time DESC LIMIT 5;
   ```
3. 确认密码至少6位

---

### ❓ 页面显示空白

**问题**: 登录后页面空白或404

**解决方案**:
1. 检查浏览器控制台是否有错误
2. 确认路由配置已更新
3. 清除浏览器缓存重试
4. 检查localStorage中的role值

## 💡 开发提示

### 修改端口

**后端端口** (默认8080):
编辑 `demo/src/main/resources/application.properties`:
```properties
server.port=8080
```

**前端端口** (默认5173):
编辑 `vue-project/vite.config.ts`:
```typescript
export default defineConfig({
  server: {
    port: 5173
  }
})
```

### 热更新

- **前端**: 修改代码后自动刷新
- **后端**: 需要重启服务（或配置Spring DevTools）
